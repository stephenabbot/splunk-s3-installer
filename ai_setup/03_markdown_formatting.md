# Markdown Formatting Requirements

## Content Restrictions

### What Not to Include
- No code snippets or implementation code
- No fenced code blocks without language identifier
- No bold text using double asterisks
- No blank lines immediately after headings

### Content Approach
- Rely on document outline structure for hierarchy
- Detail what is to be used and how it is to be used
- Explain concepts, not implementations
- Use descriptive language instead of showing code

## Spacing and Structure

### Heading Spacing
- No blank lines after headings
- Single blank line between subsections
- Double blank line between major sections
- Headings followed immediately by content or next-level heading

### Paragraph and List Formatting
- No blank lines between consecutive paragraphs in same section
- Use dash for all unordered list items at all levels
- No blank lines between list items
- 2-space indentation for nested list levels
- Blank line before each list
- Blank line after each list

### Fenced Code Blocks
- Every fenced code block must have language identifier
- Format: ```language not ```
- Valid identifiers: bash, python, json, yaml
- Never use unmarked fences

## Document Organization

### Structure Requirements
- Clear hierarchical heading structure
- Consistent section ordering
- Logical information flow
- Related content grouped together
- Hard line breaks for paragraph flow
- Consistent spacing patterns throughout
