# Session Management Protocol

## Session Initialization

### Context Establishment Process
- Share initial understanding - specs/implementation files/mix
- Refine aspects - discuss ideas for solution design
- Establish general understanding - what project intends to do
- Q&A session - challenge/consider/decide design and implementation details
- Goal: reach "no more questions" before implementation

### Quality Gate Enforcement
- Review response against prevention checklist
- Identify any missed items before proceeding
- Verify all requirements understood before acting
- Confirm authority explicitly before execution

## Prevention Checklist

Before responding to any request verify:

- Have I read and understood ALL requirements stated so far?
- Am I at correct workflow stage for this response?
- Am I making assumptions that should be verified?
- Have I been asked to provide this output or am I volunteering?
- Does my response include actual work or just offers to do work?
- If ambiguous, have I asked clarifying questions?
- Am I including unnecessary elements?
- Does error handling include explanation and troubleshooting steps?
- Am I automating everything that can be automated?
- Have I verified output against style/format requirements?

## Success Metrics

Effective sessions result in:

- Reduced iteration cycles to reach acceptable output
- Fewer instances of "please provide updated X"
- Agent asks clarifying questions proactively
- Agent waits for explicit instructions to proceed
- Comprehensive error handling with guidance
- Specifications remain abstract and implementation-free
- Agent acknowledges uncertainty rather than guessing
